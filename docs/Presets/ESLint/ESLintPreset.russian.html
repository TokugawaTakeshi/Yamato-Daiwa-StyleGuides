ESLint

Общие правила для TypeScript и JavaScript

Валидность ECMAScript-кода

Стандарт ECMA-Script является довольно либеральным, позволяя писать бессмысленный код (например, можно в классе объявить геттер, который ничего не возвращает) без ошибок при его выполнении.

array-callback-return

Array имеет несколько методов для фильтрации, маппинга и сворачивания. Если мы забудем указать инструкцию возврата значения в функциях обратного вызова, которые передаются через параметры этих методов, то, возможно, это ошибка. Если вы не хотите возвращать значение или не нуждаетесь в возвращённом значение, то тогда лучше использовать метод .forEach.

Array has several methods for filtering, mapping, and folding. If we forget to write return statement in a callback of those, it’s probably a mistake. If you don’t want to use a return or don’t need the returned results, consider using .forEach instead.

情報源Официальная документация ESLint

При использовании языка TypeScript с качественным оповещением об ошибках подобного рода странности будут сразу же обнаружены, поскольку все стандартные методы (в том числе и массивов) имеют качественные определения TypeScript-типов. 

{ "array-callback-return": "error" }

constructor-super

Конструкторы унаследованных классов должны вызывать super(). Конструкторы классов, которые не унаследованы, super() вызывать не должны. Если это не замечено (программистом), то среда выполнения JavaScript выбросит ошибку выполнения кода.

Constructors of derived classes must call super(). Constructors of non derived classes must not call super(). If this is not observed, the JavaScript engine will raise a runtime error.

情報源Официальная документация ESLint

Если в конструкторе унаследованного класса обратиться к контекстной переменной this, то язык TypeScript оповестит об ошибке TS2377. Что касается JavaScript, то без данного ESLint-правила это ошибку способны заметить лишь лучшие интегрированные среды разработки, такие как IntelliJ IDEA. 

{ "constructor-super": "error" }

getter-return

The get syntax binds an object property to a function that will be called when that property is looked up. It was first introduced in ECMAScript 5. Note that every getter is expected to return a value.

情報源Официальная документация ESLint

Если код написан на TypeScript, то компилятор (вернее сказать, транспайлер) сразу же заметит это в виде ошибки TS2378. Если же код написан на JavaScript, то даже такие передовые интегрированные среды разработки, как IntelliJ IDEA не поднимут тревоги (проверено на версии 2024.1.1). 

{ "getter-return": "error" }

Предотвращение алгоритмических ошибок

В отличие от предыдущей группы правил, нарушения данной могут быть представлены валидным кодом, однако с высокой вероятностью являются алгоритмической ошибкой. Данные правила также достаточно хорошо прокомментированы официальной документацией, для многих из них мы ограничимся ещё переводом с улучшением стилистики.

for-direction

A for loop with a stop condition that can never be reached, such as one with a counter that moves in the wrong direction, will run infinitely. While there are occasions when an infinite loop is intended, the convention is to construct such loops as while loops. More typically, an infinite for loop is a bug.

情報源Официальная документация ESLint
{ "for-direction": "error" }
